# Vendored pipeline data

Offline reference data the pipeline reads at runtime. **No network access is needed at
runtime** — these files are committed artifacts.

| File | What | Provenance |
|---|---|---|
| `us_zips.csv` | ~41.7k rows of `zip,state,lat,lon` for the 50 states + DC | Generated by `build_us_zips.py` from the public-domain [`zipcodes`](https://pypi.org/project/zipcodes/) dataset (US Census / USPS derived) |
| `build_us_zips.py` | The generator (audit trail, **not** a runtime dependency) | — |

`pipeline/regions.py` loads `us_zips.csv` to build a `Region` per state (bbox + center
**derived** from the state's ZIP coordinates) plus a synthesized national `usa` region. This is
the "coverage is data, not code" model — adding/adjusting coverage is a data change, not a code
change.

## Regenerate

The CSV is committed, so you normally never touch it. To rebuild from source (writes the CSV in
place; installs nothing on the host):

```bash
docker run --rm -v "$PWD:/src" -w /src python:3.12-slim \
  sh -c "pip install -q zipcodes && python pipeline/data/build_us_zips.py"
```

Territories (PR/GU/VI/AS/MP, the FM/MH/PW compacts) and military APO/FPO ranges (AA/AE/AP) are
intentionally excluded — the donation sources OpenDrop sweeps don't list bins or stores there.
